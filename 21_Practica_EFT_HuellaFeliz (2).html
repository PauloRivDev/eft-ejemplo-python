¿Qué vamos a construir?
CONTEXTO DEL PROBLEMA

La veterinaria "HuellaFeliz"

Imagina que trabajas para una veterinaria pequeña. Antes de tener este programa, la veterinaria anotaba en papel el nombre de cada mascota, su especie, edad, peso y si estaba esterilizada. Ese papel se perdía, era difícil de buscar y no se podía actualizar rápido.

Tu trabajo es crear un programa en Python que reemplace ese papel. El programa debe permitirle al empleado de la veterinaria:

  • Agregar una mascota nueva al registro.
  • Buscar una mascota por nombre para ver sus datos.
  • Eliminar una mascota del registro (por ejemplo, si se fue de cliente).
  • Actualizar automáticamente cuáles mascotas necesitan control geriátrico (las de 5 años o más).
  • Mostrar todas las mascotas registradas.
🤔 Antes de programar, piensa como usuario El empleado de la veterinaria no sabe nada de Python. Solo verá un menú con números. Cuando elige una opción, el programa le pide los datos, los valida y le avisa si cometió un error. Tú, como programador, debes asegurarte de que el programa sea a prueba de errores humanos.

¿Cómo almacena los datos el programa?
ESTRUCTURA DE DATOS

Lista de diccionarios — el "archivador" del programa

Para guardar los datos de todas las mascotas usamos una lista de diccionarios. Antes de escribir código, debes entender bien qué significa esto.

¿Qué es un diccionario en este contexto?

Un diccionario es como una ficha de registro para una mascota. Tiene campos con nombre (las claves) y los datos de esa mascota (los valores). Cada mascota tiene su propia ficha:

{
    "nombre" : "Fido",
    "especie" : "Perro",
    "edad" : 7,
    "peso" : 12.5,
    "esterilizado": True,
    "atendido" : False
}

¿Qué es la lista?

La lista es el archivador que guarda todas las fichas juntas. Cuando el programa parte, la lista está vacía ([]). Cada vez que el usuario elige "Agregar mascota", se crea una ficha nueva y se mete al archivador.

# listaMascotas con 2 mascotas registradas
[
    { "nombre": "Fido", "especie": "Perro", "edad": 7, ... },
    { "nombre": "Luna", "especie": "Gato", "edad": 3, ... }
]

Detalle de cada campo

CampoTipo¿Qué significa?Restricción
"nombre"strNombre de la mascotaNo vacío ni solo espacios
"especie"strTipo de animalSolo "Perro", "Gato" o "Ave"
"edad"intAños de vidaEntero mayor a 0
"peso"floatPeso en kilogramosNúmero mayor a 0.0 (puede ser decimal)
"esterilizado"bool¿Está esterilizado?El usuario escribe "s" o "n"; el programa lo convierte a True/False
"atendido"bool¿Requiere control geriátrico?Siempre inicia en False. Se actualiza automáticamente con la opción 4
🔵 ¿Por qué "atendido" inicia siempre en False? Cuando una mascota se registra por primera vez, aún no sabemos si necesita control. El programa aplica la regla de forma masiva solo cuando el usuario elige la opción 4. Por eso al crear la ficha siempre ponemos "atendido": False — la condición real se calcula después, no en el momento del registro.
🔵 ¿Por qué "esterilizado" se guarda como True/False y no como "s"/"n"? La letra "s" o "n" es lo que el usuario escribe, pero internamente el programa trabaja con valores booleanos (True / False) porque son más precisos y fáciles de comparar. El programa convierte la letra a bool en el momento de crear la ficha.

¿Cómo funciona el programa?
FLUJO GENERAL

El ciclo de vida del programa

El programa completo funciona como un ciclo que se repite hasta que el usuario elige "Salir". En cada vuelta del ciclo pasan exactamente tres cosas:

📋
1. Mostrar menú — mostrarMenu() imprime las 6 opciones en pantalla.
↓
⌨️
2. Leer opción — leerOpcion() espera que el usuario escriba un número del 1 al 6. Si escribe algo inválido, vuelve a pedir sin explotar.
↓
⚙️
3. Ejecutar la acción — según el número elegido, se llama a la función correspondiente (agregar, buscar, eliminar, etc.).
↓ (vuelve al inicio, hasta que opcion == 6)
# Así se ve el ciclo principal en código
while opcion != 6:
    mostrarMenu()           # paso 1
    opcion = leerOpcion()   # paso 2

    if opcion == 1:          # paso 3
        agregarMascota(listaMascotas)
    elif opcion == 2:
        buscarYMostrarMascota(listaMascotas)
    elif opcion == 3:
        eliminarMascota(listaMascotas)
    elif opcion == 4:
        actualizarEstados(listaMascotas)
    elif opcion == 5:
        mostrarMascotas(listaMascotas)
📌 Observa que la lista se pasa como parámetro Nota que en cada llamada se escribe agregarMascota(listaMascotas), no solo agregarMascota(). La lista se pasa como parámetro para que cada función pueda leerla o modificarla. Sin este parámetro, la función no sabría dónde están guardadas las mascotas.
💡 ¿Por qué no hay elif para la opción 6? Porque cuando opcion == 6, la condición del while (opcion != 6) se vuelve False y el ciclo termina por sí solo. No necesitas manejar el "Salir" con un bloque de código — el propio while se encarga.

Un sistema de este tipo siempre necesita las mismas categorías de funciones:

🔵 ¿Por qué estas dos variables van fuera de todo?
  • listaMascotas = [] es el "archivador vacío" donde se guardarán todas las fichas. Existe desde que el programa parte y sobrevive durante toda la ejecución. Si lo declararas dentro de una función, cada vez que se llame esa función empezaría vacío de nuevo y se perdería todo lo ingresado.

2
leerOpcion() — No recibe nada, retorna int validado (1 a 6) 
def leerOpcion():
    opcionTexto     = ""
    opcionIngresada = 0

    while True:
        opcionTexto = input("Ingrese su opcion: ")

        if opcionTexto.isdigit():
            opcionIngresada = int(opcionTexto)

            if opcionIngresada >= 1 and opcionIngresada <= 6:
                break
            else:
                print("Opcion fuera de rango. Ingrese un numero entre 1 y 6.")
        else:
            print("Opcion invalida. Ingrese solo numeros.")

    return opcionIngresada
📌 El patrón while True + isdigit() + break — léelo así
  1. El ciclo entra siempre (while True nunca falla).
  2. Se pide un texto con input(). El resultado es siempre texto, aunque el usuario escriba "3".
  3. isdigit() verifica que ese texto sea solo dígitos (0-9). Bloquea letras, puntos, guiones y espacios.
  4. Si pasa la prueba, se convierte a número entero con int() y se verifica el rango 1–6.
  5. Si el rango está bien, break sale del ciclo.
  6. return entrega el número ya validado al código que llamó a esta función.
❌ El error más grave: convertir antes de validar Nunca escribas int(input("Ingrese su opcion: ")). Si el usuario escribe "hola", Python intenta convertir esa letra a número y el programa se cae con un error. Siempre: primero input(), luego isdigit(), y recién después int().

Funciones de validación
PATRÓN OBLIGATORIO

Reciben un dato, retornan True o False. Nunca imprimen mensajes.

Las funciones de validación tienen una regla muy importante: su único trabajo es revisar si un dato es válido y retornar True o False. Los mensajes de error al usuario no van aquí — van en agregarMascota, que es quien llama a estas funciones.

🤔 ¿Por qué separar "validar" de "mostrar el error"? Imagina que en el futuro quieres reutilizar validarNombre en otra parte del programa. Si el mensaje de error estuviera dentro de la función, aparecería en todos lados aunque no quisieras. Al separar responsabilidades, la función valida y quien la llama decide qué mensaje mostrar. Es más flexible y más fácil de mantener.
📌 Patrón de validación — siempre igual 
def validarAlgo(valor):
    esValido = False          # arranca asumiendo que es inválido
    if <condición cumplida>:
        esValido = True      # solo cambia si cumple la regla
    return esValido

validarNombre(nombre) — No vacío ni solo espacios
def validarNombre(nombre):
    esValido = False

    if nombre.strip() != "":
        esValido = True

    return esValido
💡 ¿Para qué sirve strip()? strip() elimina los espacios del inicio y del final de un texto. Si el usuario presiona Enter sin escribir nada, obtienes "". Si presiona varios espacios y Enter, obtienes " ". Sin strip(), ese segundo caso pasaría la validación porque " " != "" es True. Con strip(), se elimina y queda "", que sí falla correctamente.
4
validarEspecie(especie) — Solo Perro, Gato o Ave
def validarEspecie(especie):
    esValido = False

    especieNormalizada = especie.strip().capitalize()

    if especieNormalizada == "Perro" or especieNormalizada == "Gato" or especieNormalizada == "Ave":
        esValido = True

    return esValido
💡 capitalize() hace la comparación justa para el usuario Sin capitalize(), si el usuario escribe "perro" o "PERRO", la comparación con "Perro" falla. Con capitalize(), todo se convierte a "primera letra mayúscula, resto minúscula" antes de comparar. Así el usuario puede escribir de cualquier forma y el programa lo acepta.
5
validarEdad(edad) — Entero mayor a 0
def validarEdad(edad):
    esValido = False

    if edad.isdigit():
        edad = int(edad)

        if edad > 0:
            esValido = True

    return esValido
📌 Dos validaciones en cadena Primero isdigit() verifica que el texto sea un número entero (bloquea letras, punto decimal, negativos). Solo si esa prueba pasa, se convierte con int() y se verifica que sea mayor a 0. Las dos condiciones deben cumplirse.
6
validarPeso(peso) — Decimal mayor a 0.0
def validarPeso(peso):
    esValido = False

    try:
        peso = float(peso)

        if peso > 0.0:
            esValido = True

    except ValueError:
        esValido = False

    return esValido
🔵 ¿Por qué aquí uso try/except y no isdigit()? isdigit() revisa carácter por carácter y falla si encuentra un punto. El texto "12.5" tiene un punto, así que "12.5".isdigit() retorna False, aunque sea un peso perfectamente válido. Para decimales siempre se usa try: float(x). Si el texto no puede convertirse a decimal (por ejemplo "hola"), Python lanza un ValueError y el bloque except lo captura sin que el programa se caiga.

validarEsterilizado(respuesta) — "s" o "n"
def validarEsterilizado(respuesta):
    esValido = False

    respuestaNormalizada = respuesta.strip().lower()

    if respuestaNormalizada == "s" or respuestaNormalizada == "n":
        esValido = True

    return esValido
💡 lower() para no castigar al usuario por mayúsculas lower() convierte todo a minúsculas. Así "S", "s" o "S " quedan todos como "s" antes de comparar. El usuario puede escribir como quiera y el programa lo acepta.

buscarMascota(listaMascotas, nombre) — Retorna la posición o -1

Esta función recorre la lista buscando una mascota por nombre. Si la encuentra, retorna en qué posición está (0, 1, 2…). Si no la encuentra, retorna -1. El número -1 funciona como una señal de "no encontrado" porque no puede ser una posición real en una lista.

def buscarMascota(listaMascotas, nombre):
    posicion   = -1       # resultado por defecto: no encontrado
    encontrado = False    # bandera: se vuelve True al encontrar
    indice     = 0        # posición actual en la lista

    while indice < len(listaMascotas) and encontrado == False:

        if listaMascotas[indice]["nombre"].lower() == nombre.strip().lower():
            posicion   = indice
            encontrado = True

        indice = indice + 1

    return posicion
📌 Las 3 variables obligatorias — entiende qué hace cada una
  • posicion = -1: el resultado que se devolverá si la búsqueda termina sin encontrar nada. Si se encuentra, se actualiza al índice real.
  • encontrado = False: controla el while. Cuando se halla la mascota, cambia a True y el ciclo para. Sin esto, el ciclo seguiría buscando innecesariamente después de encontrar.
  • indice = 0: el "cursor" que avanza de posición en posición. Empieza en 0 (primera ficha) y sube de uno en uno.
💡 El while tiene dos condiciones y ambas deben cumplirse indice < len(listaMascotas) — evita que el cursor se salga de la lista y cause un error. encontrado == False — detiene la búsqueda en cuanto se halla. Las dos condiciones van unidas con and: si alguna falla, el ciclo para.
🔵 ¿Por qué esta función no imprime nada? Porque es llamada por dos funciones distintas: buscarYMostrarMascota (que muestra un mensaje de éxito) y eliminarMascota (que muestra un mensaje de eliminación). Si los mensajes estuvieran aquí, no podrías controlar qué decir en cada caso. Al retornar solo la posición, cada función decide qué hacer con ese resultado.
❌ El error más frecuente: return dentro del while Si pones return posicion dentro del while, la función termina al revisar el primer elemento sin haber buscado en el resto. El return va siempre fuera del ciclo, después de que termine toda la búsqueda.

agregarMascota(listaMascotas) — Pide, valida y agrega la ficha

Esta es la función más larga. Su trabajo: pedir todos los datos al usuario, llamar a cada función de validación, y si todos son válidos, crear el diccionario y agregarlo a la lista.

def agregarMascota(listaMascotas):
    # 1. Pedir todos los datos primero
    nombre       = input("Ingrese nombre de la mascota: ")
    especie      = input("Ingrese especie (Perro / Gato / Ave): ")
    edad         = input("Ingrese edad en anios (entero mayor a 0): ")
    peso         = input("Ingrese peso en kg (numero mayor a 0): ")
    esterilizado = input("Esta esterilizado? (s/n): ")

    # 2. Validar cada dato con su función
    nombreValido       = validarNombre(nombre)
    especieValida      = validarEspecie(especie)
    edadValida         = validarEdad(edad)
    pesoValido         = validarPeso(peso)
    esterilizadoValido = validarEsterilizado(esterilizado)

    # 3. Verificar y mostrar el primer error encontrado
    if nombreValido == False:
        print("Error: el nombre no puede estar vacio.")
    elif especieValida == False:
        print("Error: la especie debe ser Perro, Gato o Ave.")
    elif edadValida == False:
        print("Error: la edad debe ser un numero entero mayor a 0.")
    elif pesoValido == False:
        print("Error: el peso debe ser un numero mayor a 0.")
    elif esterilizadoValido == False:
        print("Error: debe ingresar 's' para si o 'n' para no.")
    else:
        # 4. Solo si TODO es válido, crear la ficha y agregarla
        nuevaMascota = {
            "nombre"      : nombre.strip(),
            "especie"     : especie.strip().capitalize(),
            "edad"        : int(edad),
            "peso"        : float(peso),
            "esterilizado": esterilizado.strip().lower() == "s",
            "atendido"    : False
        }
        listaMascotas.append(nuevaMascota)
        print("Mascota agregada correctamente.")
🤔 ¿Por qué if/elif y no varios if? Con elif, si el nombre falla, el programa muestra solo ese error y para. Con múltiples if independientes, el programa mostraría todos los errores a la vez, lo que confunde al usuario. En un menú, es mejor mostrar un error a la vez.
⚠️ El diccionario nuevo debe tener un nombre diferente al parámetro El parámetro se llama listaMascotas. El diccionario nuevo se llama nuevaMascota. Si los nombraras igual, sobreescribirías la referencia a la lista y luego append no agregaría nada donde debería. Siempre un nombre diferente.
💡 "esterilizado": esterilizado.strip().lower() == "s" Esta línea convierte directamente el texto "s"/"n" en un booleano. Si el usuario escribió "s" (o "S", o " s "), la expresión retorna True. Si escribió "n", retorna False. No necesitas un if extra para esto.

buscarYMostrarMascota(listaMascotas) — Pide nombre, muestra datos
def buscarYMostrarMascota(listaMascotas):
    nombre   = input("Ingrese nombre de la mascota a buscar: ")
    posicion = buscarMascota(listaMascotas, nombre)

    if posicion != -1:
        mascota = listaMascotas[posicion]
        print("--- Mascota encontrada ---")
        print("Nombre:       " + mascota["nombre"])
        print("Especie:      " + mascota["especie"])
        print("Edad:         " + str(mascota["edad"]) + " anios")
        print("Peso:         " + str(mascota["peso"]) + " kg")
        print("Esterilizado: " + ("Si" if mascota["esterilizado"] else "No"))
        print("Atendido:     " + ("Si" if mascota["atendido"] else "No"))
    else:
        print("Mascota no encontrada.")
💡 ¿Qué significa "Si" if mascota["esterilizado"] else "No"? Es una forma corta de escribir: si el campo es True, mostrar "Si"; si es False, mostrar "No". Lo necesitamos porque no podemos hacer print("Esterilizado: " + True) — Python no puede concatenar texto con booleano directamente.
11
eliminarMascota(listaMascotas) — Busca y elimina con pop()
def eliminarMascota(listaMascotas):
    nombre   = input("Ingrese nombre de la mascota a eliminar: ")
    posicion = buscarMascota(listaMascotas, nombre)

    if posicion != -1:
        listaMascotas.pop(posicion)
        print("Mascota eliminada correctamente.")
    else:
        print("Mascota no encontrada.")
📌 pop(posicion) elimina el elemento en esa posición pop(0) elimina el primero, pop(1) el segundo, etc. Como ya tenemos la posición exacta de buscarMascota, se la pasamos directamente. No reimplementamos la búsqueda adentro — la reutilizamos.

actualizarEstados(listaMascotas) — Aplica la regla a TODA la lista

Esta función recorre todos los registros sin excepción y aplica la regla: si la mascota tiene 5 o más años, necesita control geriátrico (atendido = True). Si tiene menos de 5, no lo necesita (atendido = False).

def actualizarEstados(listaMascotas):
    indice = 0

    while indice < len(listaMascotas):

        if listaMascotas[indice]["edad"] >= 5:
            listaMascotas[indice]["atendido"] = True
        else:
            listaMascotas[indice]["atendido"] = False

        indice = indice + 1   # SIEMPRE fuera del if/else

    print("Estados de atencion actualizados.")
🤔 ¿Por qué usa while y no dos condiciones como buscarMascota? buscarMascota para cuando encuentra — por eso usa encontrado == False en la condición. actualizarEstados no puede parar — debe revisar y actualizar absolutamente todas las mascotas. Por eso usa solo indice < len(lista): sigue hasta el final sin excepciones.
❌ Error muy frecuente: indice dentro del if o del else Si pones indice = indice + 1 dentro del if, cuando la edad es mayor a 5 el índice nunca avanza → el programa queda pegado revisando la misma mascota para siempre (bucle infinito). El incremento va siempre fuera del if/else para que avance en todos los casos.

mostrarMascotas(listaMascotas) — Recorre y muestra todo el registro
def mostrarMascotas(listaMascotas):
    if len(listaMascotas) == 0:
        print("No hay mascotas registradas.")
    else:
        indice = 0
        print("-------- MASCOTAS REGISTRADAS --------")

        while indice < len(listaMascotas):
            mascota = listaMascotas[indice]
            print("Nombre:       " + mascota["nombre"])
            print("Especie:      " + mascota["especie"])
            print("Edad:         " + str(mascota["edad"]) + " anios")
            print("Peso:         " + str(mascota["peso"]) + " kg")
            print("Esterilizado: " + ("Si" if mascota["esterilizado"] else "No"))
            print("Atendido:     " + ("Si" if mascota["atendido"] else "No"))
            print("--------------------------------------")
            indice = indice + 1
💡 Verificar si la lista está vacía antes de recorrer Si la lista está vacía y entras directo al while, el ciclo no ejecuta ninguna vuelta (porque 0 < 0 es False) — no pasa nada malo. Pero el mensaje "No hay mascotas" es mejor experiencia de usuario que un silencio. Siempre verifica primero.

Escribir código de acción fuera de un def El código se ejecuta una sola vez al iniciar, no cuando el usuario lo pide Todo va dentro de un def. Solo el while va afuera
int(input(...)) sin validar antes Si el usuario escribe texto, el programa se cae con un error no controlado Siempre: input() → isdigit() → int()
Usar el mismo nombre para la lista y el dict nuevo listaMascotas.append(listaMascotas) — agrega la lista dentro de sí misma El dict nuevo siempre tiene un nombre diferente: nuevaMascota
return posicion dentro del while en buscarMascota La función retorna después del primer elemento sin revisar el resto El return va siempre fuera del while
indice = indice + 1 dentro del if en actualizarEstados Bucle infinito cuando la condición se cumple El incremento va fuera del if/else, siempre
Imprimir mensajes dentro de las funciones validar... La función tiene doble responsabilidad y no se puede reutilizar Las validaciones solo retornan True/False. Los mensajes van en agregarMascota
posicion != 1 en lugar de posicion != -1 pop(1) elimina el segundo elemento de la lista aunque no sea el buscado La señal de "no encontrado" es siempre -1, no 1
